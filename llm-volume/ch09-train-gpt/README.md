# 第九章 · 训练你的 GPT

> 上一章的 GPT「能跑但不会说话」——参数是随机的,生成全是乱码。这一章给它喂真实文字,
> 用梯度下降反复调参,**亲眼看着 loss 从瞎猜水平一路下降,生成从乱码变成像样的句子**。
>
> 为了能在普通电脑上几十秒跑完,我们用一小段中文语料(几首古诗),模型也很小。
> 目标是**看清训练这件事本身**——它和训练 GPT-4 的原理一模一样,只是规模差了上亿倍。

---

## 1. 准备语料:字符级词表

我们用四首耳熟能详的古诗当语料(重复几遍,让小模型有机会学会)。**字符级**分词最简单:
每个不同的字给一个编号。

```python
chars = sorted(set(TEXT))
vocab_size = len(chars)
stoi = {c: i for i, c in enumerate(chars)}   # 字 -> 编号
itos = {i: c for i, c in enumerate(chars)}   # 编号 -> 字
encode = lambda s: [stoi[c] for c in s]
decode = lambda l: "".join(itos[i] for i in l)
data = torch.tensor(encode(TEXT), dtype=torch.long)
```

```text
语料总字数: 400  不同字数(词表): 74
前 24 个字: 床前明月光，疑是地上霜。举头望明月，低头思故乡。
```

---

## 2. 构造训练样本:x 和「右移一位」的 y

GPT 学的是「看前文,猜下一个字」。所以标签 `y` 就是输入 `x` **右移一位**——
每个位置的答案,正是它后面那个字。

```python
def get_batch():
    ix = torch.randint(0, len(data) - block_size - 1, (batch_size,))
    x = torch.stack([data[i:i + block_size] for i in ix])
    y = torch.stack([data[i + 1:i + 1 + block_size] for i in ix])  # 右移一位
    return x, y
```

举例:若 `x = 床前明月`,则 `y = 前明月光`。模型在「床」处要预测「前」,在「前」处预测「明」……
因果掩码(第六章)保证它预测某个位置时看不到答案。

---

## 3. 训练循环:四步一圈

训练就是反复做这四步,这是所有深度学习训练的通用骨架:

```python
model = GPT(vocab_size, block_size, d_model, n_head, n_layer)
opt = torch.optim.AdamW(model.parameters(), lr=3e-3)

for it in range(1, n_iters + 1):
    x, y = get_batch()           # ① 取一批数据
    _, loss = model(x, y)        # ② 前向:算预测和 loss
    opt.zero_grad()              # ③ 清空上一轮的梯度
    loss.backward()              #    反向:求每个参数的梯度
    opt.step()                   # ④ 优化器:顺着梯度挪一小步
```

> 代码见 [`code/train.py`](code/train.py)(模型用的就是第八章那个 GPT,内联在同一文件)

## 4. 看 loss 真的降下去

实跑(模型 16 万参数,600 步,CPU 上约 5 秒):

```text
参数量: 160,832

开始训练(看 loss 往下走):
  第    1 步  loss = 4.503
  第  100 步  loss = 0.037
  第  200 步  loss = 0.020
  第  300 步  loss = 0.026
  第  400 步  loss = 0.022
  第  500 步  loss = 0.022
  第  600 步  loss = 0.026
```

第 1 步 loss = 4.503,正是「瞎猜」水平(≈ ln 74 ≈ 4.30)。100 步后骤降到 0.037——
模型迅速学会了这几首诗的规律。

> 注意:这里 loss 降到接近 0,是因为**数据太小、模型相对够大**,模型基本是把诗「背」下来了
> (过拟合)。真实训练用海量数据,loss 不会、也不该降到这么低——那样反而是记硬盘,不会举一反三。
> 这里我们要的就是「背下来」,好让生成效果一眼可见。

## 5. 生成对比:乱码 → 成诗

最直观的验证,是看训练前后生成的差别。同样从「床」字起头:

**训练前**(参数随机,纯乱码):

```text
床锄霜中滴不入下滴风里前黄白山盘下疑海下地，谁思欲河地处禾流闻花头床啼楼知山声。光
```

**训练后**(从「床」字续写):

```text
床前明月光，疑是地上霜。举头望明月，低头思故乡。
春眠不觉晓，处处闻啼鸟。夜来风雨声，花落知多少。
```

**训练后**(从「春」字续写):

```text
春眠不觉晓，处处闻啼鸟。夜来风雨声，花落知多少。
白日依山尽，黄河入海流。欲穷千里目，更上一层楼。
```

给一个起头字,它就能顺着背出整首诗,甚至接着背下一首——一个真正「学会」了语料的 GPT,
是我们自己从零搭起来、从零训出来的。

## 本章小结

- **训练 = 四步循环反复跑**:取 batch → 前向算 loss → 反向求梯度 → 优化器挪一步。这是所有深度学习的通用骨架。
- **数据构造**:标签 `y` 是输入 `x` **右移一位**,每个位置的答案就是它的下一个字。
- **loss 从 ln(vocab) 附近一路下降**:从「均匀瞎猜」逐渐学会语料规律。第 1 步 4.503,100 步后 0.037。
- **生成对比最能说明问题**:训练前乱码,训练后能背出整首诗。
- 小数据 + 小模型会**过拟合(背下来)**,这里正好用来让效果一眼可见;真实训练靠海量数据避免死记。
- **和训练 GPT-4 原理完全相同**,区别只在数据规模(整个互联网)、参数规模(上千亿)、算力(几个月)。

下一章,我们专门研究**怎么从模型嘴里「取词」**——`temperature`、`top-k`、`top-p` 这些
采样策略,决定了生成是保守复读还是天马行空。
