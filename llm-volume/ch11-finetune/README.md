# 第十一章 · 微调(LoRA)

> 前面我们从零训出了一个 GPT。但现实中你几乎不会从零训大模型——成本太高。
> 更常见的是:拿一个**已经训好的**大模型,用很小的代价让它适配你自己的任务。这就是**微调**。
>
> 这一章聚焦当下最流行的微调方法 **LoRA(Low-Rank Adaptation)**:
> 它冻结原模型,只训练一个**极小**的旁路,就能让大模型学会新任务。我们会手写一个能跑的最小 LoRA。

---

## 1. 全参微调的痛点

最直接的微调叫**全参微调(full fine-tuning)**:把模型所有权重都拿来继续训练。问题是:

- 一个权重矩阵动辄 4096×4096,几千万个数。要训练、要存梯度、要存优化器状态——**显存爆炸**。
- 每个任务都得存一整份新权重。十个任务,十份几十 GB 的模型。

有没有办法,**只动一点点参数**就让模型适配新任务?

---

## 2. LoRA 的核心思想:低秩增量

LoRA 的关键观察:微调时,权重的**改动量** ΔW 往往不需要那么「丰富」——
它可以用一个**低秩矩阵**很好地近似。

```text
原本:  W' = W + ΔW         ΔW 和 W 一样大 (d×d),要学 d² 个数
LoRA:  W' = W + B·A         A 是 (r×d),B 是 (d×r),r 很小
              └冻结┘ └可训练┘  只学 2·d·r 个数,r<<d 时省得惊人
```

省多少?直接看账本:

```text
矩阵 4096×4096:  全参微调 16,777,216 个  |  LoRA(r=4) 32,768 个  |  仅为 0.20%
矩阵 4096×4096:  全参微调 16,777,216 个  |  LoRA(r=8) 65,536 个  |  仅为 0.39%
```

> 代码见 [`code/lora_idea.py`](code/lora_idea.py)

`r` 取个位数,可训练参数常常降到原来的 **1% 以下**。

---

## 3. 为什么「低秩」常常够用

凭什么相信 ΔW 是低秩的?造一个真实的低秩改动量,用 **SVD(奇异值分解)** 看它的「有效维度」:

```text
ΔW 的前 8 个奇异值: [7.901 5.846 5.431 0.    0.    0.    0.    0.   ]
第 4 个及以后基本为 0 -> ΔW 的'有效维度'就是 3,天生低秩。
```

再用不同的秩 `r` 去重建它,看误差:

```text
用秩 r 重建 ΔW 的相对误差:
  r=1:  相对误差 = 0.7106
  r=2:  相对误差 = 0.4836
  r=3:  相对误差 = 0.0000
  r=4:  相对误差 = 0.0000
```

**一旦 r 达到改动量的真实秩(这里是 3),误差就归零**。这就是 LoRA 的底气:
经验上,微调的改动量确实接近低秩,选对 `r` 几乎无损。

---

## 4. 手写一个最小 LoRA 层

把思想落成代码。一个 LoRA 线性层 = 冻结的 `base` + 可训练的低秩旁路:

```python
class LoRALinear(nn.Module):
    def __init__(self, d_in, d_out, r, alpha=8):
        super().__init__()
        self.base = nn.Linear(d_in, d_out, bias=False)
        self.base.weight.requires_grad = False     # 冻结预训练权重

        self.A = nn.Linear(d_in, r, bias=False)     # 降维 d_in -> r
        self.B = nn.Linear(r, d_out, bias=False)    # 升维 r -> d_out
        nn.init.zeros_(self.B.weight)               # B 初始化为 0(关键技巧)
        self.scale = alpha / r

    def forward(self, x):
        return self.base(x) + self.scale * self.B(self.A(x))
```

> 代码见 [`code/lora_train.py`](code/lora_train.py)

**为什么 B 初始化为 0?** 这样训练**起点**处旁路 `B·A = 0`,整层完全等于原模型——
不破坏已有能力,从「原样」平滑地开始适配。这是 LoRA 的一个关键小技巧。

---

## 5. 真的训练它:只有 LoRA 在学

任务设成「base + 一个秩为 R 的改动量」(正好是 LoRA 能完美拿捏的情形)。训练时只优化 A、B:

```text
===== 参数账本 =====
  冻结(base 权重)    : 1,024 个  <- 不训练
  可训练(LoRA 的 A+B): 256 个  <- 只训这些
  可训练占比          : 20.0%

===== 训练(只更新 LoRA)=====
  第   1 步  loss = 26.9052
  第 100 步  loss = 0.3689
  第 200 步  loss = 0.0000
  第 400 步  loss = 0.0000

base 权重训练后是否完全没变: True  <- 冻结生效
LoRA 的 B 是否已从 0 学出了内容: True
```

两件事得到印证:

1. **loss 干净地降到 0**——只靠那 256 个新参数,模型就学会了新任务。
2. **base 权重一个数都没变**(`True`)——原模型完好无损,我们只是给它挂了个小旁路。

> 本例 `D` 很小,所以可训练占比有 20%;真实大模型里这个比例是 **<1%**。

## 本章小结

- **全参微调**要更新所有权重,显存和存储成本都极高,每个任务还得存一整份模型。
- **LoRA** 冻结原权重 `W`,只额外学一个低秩增量 `B·A`(两个瘦矩阵)。可训练参数从 `d²` 降到 `2·d·r`,常常 <1%。
- **底气**:微调的改动量 ΔW 经验上接近低秩(奇异值能量集中在前几个);秩 `r` 覆盖有效维度时,低秩近似几乎无损。
- **B 初始化为 0**:训练起点处旁路为 0,整层等于原模型,不破坏已有能力。
- 实跑印证:**只训 LoRA,base 纹丝不动,loss 照样降到 0**。
- 部署时可把 `B·A` 合并回 `W`(零额外开销),也可不合并、随时插拔、**按任务切换适配器**。
