# ch03 · 语言模型的本质:猜下一个字

前两章我们准备了工具(数学直觉 + PyTorch)。从这一章开始,正式回答那个核心问题:

> **大模型到底在干什么?**

答案朴素到让人意外:

> **它在反复做一件事——看着前面的文字,猜下一个字。**

ChatGPT 写一篇文章,不是一次想好的,而是一个字一个字"猜"出来的:猜出一个字,接上去,再拿新的前文去猜下一个……如此往复。本章我们不碰神经网络,先用最朴素的"数数"做一个能跑、能生成的迷你语言模型,把这件事彻底看穿。

---

## 1. 语言模型 = 给出"下一个字的概率分布"

把"猜下一个字"说得精确一点:

> 语言模型是一个函数,**输入前面的文字,输出"下一个字是谁"的概率分布**。

注意是**概率分布**,不是斩钉截铁的答案。比如看到"天",模型可能觉得下一个字 25% 是"明"、25% 是"宇"……它给的是一组押注。

怎么得到这组概率?最笨也最好懂的办法:**数数**。我们准备一批名字当语料,数一数每个字后面都跟过哪些字:

```python
from collections import Counter
from corpus import NAMES, bigrams

after = {}   # after[前字] = Counter({后字: 次数})
for nm in NAMES:
    for a, b in bigrams(nm):
        after.setdefault(a, Counter())[b] += 1
```

看看语料里"天"后面出现过什么,再把次数除以总数变成概率:

```text
语料里 '天' 后面出现过:
   天 -> 明   1 次
   天 -> 宇   1 次
   天 -> 佑   1 次
   天 -> 华   1 次

于是模型认为 '天' 的下一个字概率是:
   P(下一个字 = 明 | 当前 = 天) = 1/4 = 0.25
   P(下一个字 = 宇 | 当前 = 天) = 1/4 = 0.25
   P(下一个字 = 佑 | 当前 = 天) = 1/4 = 0.25
   P(下一个字 = 华 | 当前 = 天) = 1/4 = 0.25
```

就这么简单。**输入一个字,输出下一个字的概率分布——这就是一个语言模型了。**
ChatGPT 做的是同一件事,只不过它的"前文"长得多、给概率的方式聪明得多。

> 我们用一个特殊的**边界符 `·`** 同时表示"开头"和"结尾"。`·天明·` 拆成 `(·→天)(天→明)(明→·)`,这样模型既能学会名字"以什么字开头",也能学会"何时收尾"。
>
> 只看前一个字来预测下一个,这种模型叫 **bigram(二元)模型**。代码见 `code/predict_next.py`。

## 2. 把数数做全,然后"生成"

对每一对 `(前字, 后字)` 都统计,就得到一张大表 `N`,`N[i, j]` = 字 `i` 后面跟字 `j` 的次数。每行归一化(除以该行总和),就是每个字的"下一字概率分布":

```python
import numpy as np
N = np.zeros((V, V))
for nm in NAMES:
    for a, b in bigrams(nm):
        N[stoi[a], stoi[b]] += 1
P = (N + 1) / (N + 1).sum(axis=1, keepdims=True)   # +1 平滑,避免 0 概率
```

有了概率,就能**生成**:从边界符出发,按概率"掷骰子"抽一个字,接上去,再用新字抽下一个,直到抽到边界符收尾:

```python
def generate(rng):
    idx = stoi["·"]
    out = []
    while True:
        idx = rng.choice(V, p=P[idx])   # 按概率抽下一个字
        ch = itos[idx]
        if ch == "·": break
        out.append(ch)
    return "".join(out)
```

跑出来(语料里**根本没有**这些名字):

```text
用统计模型生成的新名字:
   欣志轩欣
   佳龙涵燕
   嘉怡强静
   春豪思宇
   文云远桐
   ...
   海燕强小
```

模型没有"背"语料,而是学到了**字与字怎么搭**的统计规律,据此造出风格相似的新词。**这就是"生成"的本质**——按学到的概率分布,一个字一个字地采样。

> 这里的"掷骰子"就是**采样(sampling)**,ch10 会专门讲怎么把骰子掷得更好(temperature / top-k / top-p)。代码见 `code/bigram_counts.py`。

## 3. 模型好不好?用 loss 打分

我们怎么知道这个模型是不是真学到了东西,而不是瞎蒙?

办法:看它对**真实出现过的字对**给了多高的概率。模型给真实数据的概率越高,说明它越"懂"。把这些概率取对数、求平均、再取负,就是**负对数似然**,也就是后面训练 GPT 时天天用的**交叉熵损失(cross-entropy loss)**:

```python
def avg_loss(P):
    total, n = 0.0, 0
    for nm in NAMES:
        for a, b in bigrams(nm):
            total += -np.log(P[stoi[a], stoi[b]])   # 概率越高,这一项越小
            n += 1
    return total / n
```

拿我们的模型和"完全瞎猜(每个字概率都是 1/V)"对比:

```text
统计模型的平均 loss = 2.9104
瞎猜基线的平均 loss = 3.6889   (= ln V = ln40)

把 loss 还原成'有效候选数' e^loss:
    瞎猜 ≈ 40 选 1   我们的模型 ≈ 18.4 选 1
```

解读:瞎猜时模型面对 40 个字像抓阄;我们的模型把"有效候选"压到了约 18 个——它确实学到了规律。

> **记住这个数字游戏**:`loss` 越小越好,`e^loss` 可以理解成模型眼里"还剩几个字在纠结"(这个量叫**困惑度 perplexity**)。**训练 GPT,本质就是想尽办法把这个 loss 往下压。** 你在后面每一章的训练日志里都会看到它。代码见 `code/evaluate_loss.py`。

## 4. 统计模型的天花板:为什么需要神经网络

我们的 bigram 模型有个硬伤——**健忘**。它永远只记得**上一个字**,再往前的全忘了:

```text
P(下一字 | 当前='天') 前三: [('宇', 0.05), ('佑', 0.05), ('华', 0.05)]
无论'天'出现在'天明'还是'春天'里,这个分布都不变。这就是局限。
```

那"多记几个字"不就行了?问题在于**存不下**。想记住前 N 个字,统计表就要 `V^(N+1)` 个格子:

```text
看前  1 个字:V^2  = 1,600 个格子
看前  2 个字:V^3  = 64,000 个格子
看前  3 个字:V^4  = 2,560,000 个格子
看前  5 个字:V^6  = 4,096,000,000 个格子
看前 10 个字:V^11 = 419,430,400,000,000,000 个格子
```

才看 10 个字就成了天文数字,而且绝大多数格子是 0(那种组合语料里从没出现过)。这叫**维度灾难**。

解决之道,是本书后半部分的全部内容:

> **不要"查表",而要学一个会"算"概率的函数——神经网络。**

这样参数量只随上下文长度温和增长,还能在**没见过的组合**上合理猜测(泛化)。具体怎么做:
- **ch05 Embedding**:把字变成带含义的向量,相近的字共享知识(不再每个格子各管各的)
- **ch06 注意力**:让模型自己决定"该回头看前文的哪些字"
- **ch07–ch08 Transformer / GPT**:把这些拼成一个能看长上下文的强大函数

代码见 `code/limitations.py`。

## 本章小结

- **语言模型 = 看前文猜下一个字**,输出的是**概率分布**,不是唯一答案
- **生成 = 按概率反复采样**,一个字接一个字;这就是 ChatGPT 写字的方式
- **loss(交叉熵)给模型打分**,越小越好;训练就是把它压低
- **bigram 只看一个字、且会维度爆炸**——这正是我们需要神经网络的理由
