# ch01 · 数学直觉(用代码代替公式)

很多人一听"大模型需要数学"就想跑。别怕。

这一章我们**不证明任何定理**,只建立四个直觉:**向量、矩阵、概率、梯度**。
每个概念都用一小段代码让你"看见"它,而不是盯着公式发呆。

> 一句话总结本章:神经网络做的事,就是**一堆数字(向量/矩阵)的乘法和加法,外加一点点"往哪个方向调整"的判断(梯度)**。

---

## 1. 向量:一串有序的数字

向量(vector)就是一列数字,比如 `[1, 2, 3]`。

在大模型里,**每个词最终都会变成一个向量**。比如"猫"可能是 `[0.2, -0.5, 0.8, ...]`。
向量就是 AI 理解世界的"语言"。

我们能对向量做什么?最重要的是**点积(dot product)**,它能衡量两个向量有多"像":

```python
import numpy as np

a = np.array([1, 0, 1])   # 想象成"猫"的特征
b = np.array([1, 0, 1])   # 另一只"猫"
c = np.array([0, 1, 0])   # "汽车"

# 点积 = 对应位置相乘再相加
print("猫·猫 =", np.dot(a, b))   # 2,很相似
print("猫·车 =", np.dot(a, c))   # 0,不相似
```

**点积越大 = 两个向量方向越一致 = 越相似。**
记住这一点 —— 第六章的"注意力机制"本质上就是在算一堆点积。

---

## 2. 矩阵:一叠向量 / 一张数字表格

矩阵(matrix)就是把多个向量叠在一起,形成一个二维表格:

```
[[1, 2, 3],
 [4, 5, 6]]      ← 这是一个 2 行 3 列的矩阵
```

神经网络里最核心的操作就是**矩阵乘法**。你可以把它理解为:
**用一个矩阵去"变换"另一堆向量** —— 旋转、缩放、重新组合它们的信息。

```python
import numpy as np

# 3 个词,每个词是 2 维向量
words = np.array([[1, 0],
                  [0, 1],
                  [1, 1]])

# 一个"变换矩阵"(2行2列),把 2 维变换成 2 维
W = np.array([[2, 0],
              [0, 3]])

# 矩阵乘法:每个词向量都被 W 变换了一次
out = words @ W      # @ 是矩阵乘法符号
print(out)
# [[2 0]    第一个词的 x 放大2倍,y 放大3倍
#  [0 3]
#  [2 3]]
```

> 神经网络里那些"参数 / 权重",绝大多数就是这样的矩阵。
> **训练模型 = 不断调整这些矩阵里的数字,让输出越来越接近我们想要的结果。**

---

## 3. 概率:模型其实在"押注"

大模型不会斩钉截铁地说"下一个字一定是 X"。
它会给出一个**概率分布**:下一个字有 60% 可能是"猫",30% 是"狗",10% 是"鱼"……

把任意一组数字变成"加起来等于 1 的概率",用的是 **softmax**:

```python
import numpy as np

def softmax(x):
    e = np.exp(x - np.max(x))   # 减最大值是为了数值稳定,不影响结果
    return e / e.sum()

scores = np.array([2.0, 1.0, 0.1])   # 模型对三个候选字的"打分"
probs = softmax(scores)
print("概率:", probs)            # [0.659, 0.242, 0.099]
print("加起来:", probs.sum())    # 1.0
```

**softmax 把"打分"变成"概率"。** 大模型输出的最后一步永远是它。

---

## 4. 梯度:告诉我们"往哪边调"

这是最关键、也最容易被神化的概念。其实它很朴素:

> 梯度 = **斜率** = "如果我把这个数字调大一点点,结果会变好还是变坏?"

想象你蒙着眼睛站在山坡上,想走到山谷最低点。你只能用脚感受脚下**哪个方向是下坡**,然后朝那个方向迈一小步。重复很多次,就能到谷底。

- "山谷最低点" = 模型犯错最少的状态
- "脚感受的下坡方向" = 梯度
- "迈一小步" = 更新参数
- 这整个过程就叫 **梯度下降(gradient descent)**,是训练所有神经网络的核心

我们用代码找一个简单函数 `y = (x-3)²` 的最低点(显然答案是 x=3):

```python
# 目标:找到让 y=(x-3)^2 最小的 x
x = 0.0           # 随便从一个点出发
lr = 0.1          # 学习率:每步迈多大

for step in range(20):
    grad = 2 * (x - 3)    # 这就是 y 对 x 的梯度(斜率)
    x = x - lr * grad     # 朝"下坡"方向迈一步
    print(f"第{step+1:2d}步: x={x:.3f}")

# x 会一步步逼近 3
```

跑一下你会看到 x 从 0 慢慢爬到 3 附近。
**这就是训练大模型时,内部正在发生的事**,只不过 x 不是一个数,而是几十亿个数字同时在调整。

---

## 5. 把四个直觉串起来

现在把它们连成一条线,这就是大模型的工作骨架:

```text
词  →  向量(第1节)
      ↓
经过一层层矩阵变换(第2节),信息被反复加工
      ↓
最后用 softmax 输出"下一个字的概率分布"(第3节)
      ↓
和正确答案比较,算出误差
      ↓
用梯度(第4节)反向调整所有矩阵里的数字
      ↓
重复几百万次 → 模型学会了语言
```

如果这条线你能看懂个七八成,那么恭喜,**你已经掌握了理解整个大模型所需的全部数学直觉**。剩下的都是把这条线做得更精巧而已。
