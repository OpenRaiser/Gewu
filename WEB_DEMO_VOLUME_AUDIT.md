# Web 演武场逐卷分析与改造记录

目标：逐卷检查 Web 演示是否能把矩阵、向量、概率、梯度中的每个元素如何变化讲清楚，并按卷改造。

## 卷一 · 数理筑基

现状问题：

- 向量点积只显示箭头和最终 `dot`，缺少 `a[0]*b[0]`、`a[1]*b[1]` 两项如何合成。
- 矩阵乘只显示点被搬移，缺少每个输出坐标的点积公式。
- softmax 只展示柱状变化，缺少 `exp(score_i)`、`sum(exp)`、`exp_i/sum` 的逐元素链路。
- 梯度下降只展示轨迹，单步内部的 `grad`、`lr*grad`、`x_new` 不够显性。

改造目标：

- [x] 第一式：点积展示逐项乘积表和合计。
- [x] 第二式：矩阵乘展示每个输入点的新坐标如何由 W 的列逐项算出。
- [x] 第三式：softmax 展示每个候选字的 score、exp、除以总和、概率。
- [x] 第四式：梯度下降展示当前步的 `x_old -> grad -> lr*grad -> x_new`。

## 卷二 · PyTorch

现状问题：

- 形状 demo 只判断能不能乘，缺少单个输出格子如何由一行和一列点积得到。
- autograd demo 只展示曲线和梯度值，缺少计算图与链式法则反传路径。
- 训练 demo 只展示线条逐渐贴合，缺少 `w,b` 如何按梯度更新。

改造结果：

- [x] 第一式：展示 `out[0,0]` 来自 `A` 第 0 行和 `B` 第 0 列逐项乘加。
- [x] 第二式：展示 `x -> u=x-3 -> y=u^2` 的计算图，以及 `dy/dx=dy/du*du/dx`。
- [x] 第三式：展示一次 SGD 中 `w,b` 的更新公式。

## 卷三 · 语言模型

现状问题：

- bigram 分布只显示次数/概率柱，缺少 `(count+α)/sum` 的逐元素计算。
- 生成 demo 只显示抽中结果，缺少随机数如何落入累计概率区间。
- loss demo 只显示总 loss，缺少每个真实字对的 `-log(P[a][b])` 惩罚项。

改造结果：

- [x] 第一式：展示候选字概率 `P(b|a)=(count+α)/row.sum`。
- [x] 第二式：展示随机数 `r` 与被抽中字的累计概率区间。
- [x] 第三式：展示若干真实字对的 `-log(prob)` loss 项。
