> 面向想了解项目全貌、或想参与维护与扩展的读者。教程内容本身请看各章 `README.md`。

## 一、项目是什么

**格物**(取自「格物致知」:探究本质以求真知)是一个可扩展的动手学习总品牌,核心理念是
**看懂原理 → 亲手从零实现**。本仓库当前包含两个分卷:

| 分卷 | 作用 |
|------|------|
| **大模型卷** | 面向纯小白 / 在校学生的大语言模型(LLM)入门教程,从零亲手实现一个 GPT |
| **Agent 卷** | 从 single-agent harness 到 sub-agent、agent team 和 multi-agent 协同的工程化学习记录 |

其中「大模型卷」由两大部分组成,互相呼应:

| 部分 | 形态 | 作用 |
|------|------|------|
| **教程正文** | `llm-volume/` 下 14 章 Markdown + 可运行 Python(`ch00`–`ch14` + `appendix`) | 逐章讲原理,每个概念都配一段能立刻跑的代码 |
| **Web 可视化** | Vite + React 单页应用(`web/`) | 把核心概念做成可交互「演武场」,拖动数字即时看结果变化 |

两部分的数据与数值**严格对齐**:Web 里每个 demo 用到的数字,都来自对应章节 Python 的真实运行结果(详见各 demo 文件顶部注释)。

## 四、Web 前端架构

可视化前端是一个标准 Vite + React 单页应用,目录如下:

```
web/
├── index.html              # 入口 HTML(挂载 #root)
├── package.json            # 依赖与脚本(dev/build/preview)
├── vite.config.js          # Vite 配置(react 插件 + base: "./")
└── src/
    ├── main.jsx            # React 入口:createRoot 渲染 <App/>
    ├── App.jsx             # 总装配:定义分卷、卷目录与式导航
    ├── styles.css          # 全站样式(古籍秘笈风)
    ├── components/         # 通用组件(见下)
    │   ├── Codex.jsx       # 单个演武场的渲染器(三栏布局核心)
    │   ├── LiveNum.jsx     # 可拖动的行内朱字(数字)
    │   └── TracePanels.jsx # 承接关系与矩阵热力图等推导组件
    ├── lib/
    │   └── highlight.js    # 极简 Python 语法高亮
    └── demos/              # 36 个 demo + 4 个共享数据模块
        ├── *.jsx           # 每个文件导出一个 demo 契约对象
        ├── ch03corpus.js   # 共享:中文姓名语料
        ├── ch04tok.js      # 共享:字符分词 + BPE
        ├── ch05emb.js      # 共享:词表 + 嵌入表 + 余弦
        └── ch06attn.js     # 共享:注意力的 X 向量与算式
```

### 渲染链路

```
main.jsx  →  App.jsx  →  Codex.jsx  →  { LiveNum, BridgePanel, 各 demo 的 Viz }
```

- **main.jsx**:React 18 `createRoot` + `StrictMode`,引入 `App.jsx` 与 `styles.css`。
- **App.jsx**:核心是一个 `BOOKS` 数组,把分卷、卷册、每卷若干「式」逐一登记为
  `{ id, title, volumes: [{ id, name, scrolls: [{ id, label, demo }] }] }`。顶部书名 + 分卷入口 + 左侧卷目录 + 上方「式」标签,
  选中哪一式就把对应的 demo 契约对象交给 `<Codex>` 渲染。
- **Codex.jsx**:把一个 demo 契约渲染成左中右**三栏演武场**:
  - 左栏「心法·经文」:逐行展示代码(`lines`),其中 `{{朱字}}` 由 `LiveNum` 接管,可拖动;
    上方展示承前启后的来源关系,经文下方放置播放条
    (演法 / 暂停 / 续演 / 退 / 进 / 收功),按 `frames` 逐帧推进。
  - 中栏「演武场」:渲染该 demo 的 `Viz`(一张 SVG),专注展示逐帧推导结果。
  - 右栏「批注」:显示当前帧的 `note` 讲解,以及 `terms` 术语释义。

### 通用组件

| 组件 | 职责 |
|------|------|
| `LiveNum.jsx` | 行内可拖动数字。基于指针事件,**每拖 14px = 1 个 step**,按 `paramDefs` 的 min/max/step 裁剪取整,实时驱动重算。 |
| `TracePanels.jsx` | 承接关系与通用推导组件。`BridgePanel` 展示承前/本式/启后/来源,`MatrixHeatmap` 用于矩阵、权重、mask 等逐元素可视化。 |
| `Codex.jsx` | 演武场渲染器,串起代码、Viz、播放、批注、术语五块,管理 `params`/播放帧状态。 |

## 五、Demo 契约(核心设计)

每个 `demos/*.jsx` 导出**同一种形状**的对象,`Codex` 据此渲染。这是整个 Web 端的统一接口:

```js
export const xxxDemo = {
  title,       // 演武场标题,如 "演武场 · 化整为简"
  intro,       // 顶部导语(允许 HTML 加粗)
  lines,       // 代码行数组:[{ text, stage }],text 里用 {{param}} 标记朱字
  paramDefs,   // 朱字定义:{ name: { min, max, step, fmt } }
  initial,     // 朱字初值:{ name: value }
  compute,     // (p) => derived,把当前朱字算成可展示的派生量
  frames,      // (d0, d) => [{ line, stage, say }],播放脚本(逐帧高亮+讲解)
  Viz,         // ({ derived, stage }) => <svg>,中栏可视化(360×300)
  note,        // (stage, p, d) => string,右栏当前帧批注
  terms,       // [{ t, d }],术语释义(允许 HTML)
  pyCode,      // 可选:对应 Python 源码或来源摘录,当前 Web 不直接执行
  playMs,      // 每帧间隔毫秒(默认约 950)
  localCmd,    // 可选:本地复现实验命令,作为维护线索保留
};
```

### 两个关键机制

1. **朱字联动(LiveNum)**:`lines[].text` 里写 `{{w}}` 这样的占位符,`Codex` 用
   `/(\{\{\w+\}\})/g` 切分;占位符替换为可拖动的 `LiveNum`,其余按 `highlight.js` 上色。
   拖动 → 改 `params` → `compute(params)` 重算 → `Viz` 即时重绘。

2. **分阶段播放(stage)**:每行代码带一个 `stage`;`frames` 返回的每帧含
   `{ line, stage, say }`。点「演法」后按 `playMs` 逐帧推进:高亮对应代码行、
   把 `stage` 传给 `Viz` 控制「展开到第几步」、在右栏显示 `say`/`note`。
   于是「拖数字看终态」与「点演法看推导过程」两种交互并存。

> **数值真实性约定**:`compute`、`frames`、`Viz` 里出现的所有数字,都来自对应章节
> Python 脚本的**真实运行结果**,demo 文件顶部注释会标注来源(脚本名 + 实测值),
> 不允许凭空编造。

## 六、运行方式

### Web 可视化

```bash
cd web
npm install        # 首次安装依赖
npm run dev        # 本地开发(热更新)
npm run build      # 产出静态文件到 dist/
npm run preview    # 本地预览构建产物
```

### 教程 Python 示例

每章代码可独立运行,例如:

```bash
cd llm-volume/ch13-inference/code && python3 quantization.py
cd llm-volume/ch14-applications/code && python3 minimal_rag.py
```

环境准备见 `llm-volume/ch00-setup/README.md`,报错排查见 `llm-volume/appendix/`。

## 八、如何新增一个演武场 demo

1. 先在对应章节 `code/` 里把 Python 跑通,**记下真实输出数字**。
2. 在 `web/src/demos/` 新建 `xxx.jsx`,按第五节的契约导出 `xxxDemo`;
   文件顶部注释写明数字来源(脚本名 + 实测值)。
3. 在 `web/src/App.jsx` 顶部 `import { xxxDemo } from "./demos/xxx.jsx";`,
   再到对应卷的 `scrolls` 数组里加一项 `{ id, label: "第N式 · 四字名", demo: xxxDemo }`。
4. `npm run build` 验证构建通过,`npm run dev` 自查交互(拖朱字、点演法)。

命名沿用秘笈风:卷名「卷N · 四字名 (英文/技术名)」、式名「第N式 · 四字名」、
标题「演武场 · 四字名」。
