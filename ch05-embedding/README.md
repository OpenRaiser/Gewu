# ch05 · Embedding:让每个词带上"含义"

上一章我们把文字切成 token、编上号。一句话进了模型,变成一串整数。但这里藏着一个大问题:

> **编号只是"身份证号",本身不带任何含义。**

"猫"是 0 号、"狗"是 1 号、"桌子"是 2 号——可编号 0 和 1 的距离(1),和编号 1 和 2 的距离(1)一模一样。难道"猫"离"狗",跟"狗"离"桌子"一样远?显然荒唐。编号是我们**随便编**的,数值大小纯属巧合。

这一章要解决的就是:**怎么把每个词,变成一个真正能表达"含义"的东西?** 答案是 **Embedding(词嵌入)**——用一个稠密的小数向量代表每个词,并且让**意思相近的词,向量也相近**。这是从"符号"迈向"语义"的关键一步,也是后面注意力机制能work的前提。

---

## 1. 编号不是含义,one-hot 也不行

先把问题摆清楚。直接拿编号当含义,数值距离是假的:

```python
VOCAB = ["猫", "狗", "桌子", "椅子"]
stoi = {w: i for i, w in enumerate(VOCAB)}   # {'猫':0,'狗':1,'桌子':2,'椅子':3}

|猫(0) - 狗(1)|  = 1
|狗(1) - 桌子(2)| = 1   # 距离一样,但猫狗明明更近
```

一个直接的补救想法:别用大小,让每个词**各占一格**——第 i 号词就是"第 i 位是 1、其余全 0"的向量。这叫 **one-hot(独热编码)**:

```python
猫   -> [1 0 0 0]
狗   -> [0 1 0 0]
桌子 -> [0 0 1 0]
椅子 -> [0 0 0 1]
```

它确实解决了"大小是假的"问题——谁也不比谁大。但它带来两个新麻烦:

```text
距离(猫, 狗)  = 1.414
距离(猫, 桌子) = 1.414
-> 任意两个不同词的 one-hot 距离都是 √2 ≈ 1.414,完全一样。
```

- **所有词两两等距**:模型还是看不出"猫更像狗"。每个词彼此孤立,毫无远近。
- **维度爆炸**:词表多大,向量就多长。GPT 词表 5 万~10 万,one-hot 就是 5 万~10 万维,而且全是 0,极度浪费。

我们真正想要的,是一个**不太长、稠密、且能表达含义**的向量。

## 2. Embedding = 一张可查的"含义向量表"

办法朴素得不能再朴素:**准备一张表**。它有 `V` 行(词表大小)、`D` 列(每个词用 `D` 个小数表示),第 `i` 行就是第 `i` 号词的向量。这张表叫 **Embedding 矩阵**,每个词对应的那一行向量,叫它的 **embedding(词嵌入)**。

下面手填一张 4 词 × 3 维的小表,故意让"猫·狗"两行接近、"桌子·椅子"两行接近:

```python
import numpy as np
E = np.array([
    [0.9, 0.8, 0.1],   # 猫
    [0.8, 0.9, 0.2],   # 狗
    [0.1, 0.2, 0.9],   # 桌子
    [0.2, 0.1, 0.8],   # 椅子
])
```

"查"一个词的向量,就是"取出表的第 `i` 行"——`E[i]`,就这么简单。但这里有个值得点破的真相:**上一节的 one-hot 不是白费,它正是"查表"这个动作的数学写法**:

```text
【查表方式一】按行号直接取
  E[0] (猫) = [0.9 0.8 0.1]

【查表方式二】one-hot 向量 × E,得到的正是同一行
  猫的 one-hot [1 0 0 0] @ E = [0.9 0.8 0.1]
  和 E[0] 一致吗?True
```

`one-hot @ E` 和 `E[i]` 完全等价。真实代码里当然用索引(快得多),但把它理解成**矩阵乘法**,你就能看清:Embedding 其实是神经网络的**一层**,而不是什么特殊的预处理。

一个句子,就是一串编号,查表后变成**一叠向量**:

```text
句子 ['猫', '狗', '椅子'] 的编号 [0, 1, 3]
查出的向量组 shape = (3, 3)   # 3 个词 × 3 维
[[0.9 0.8 0.1]
 [0.8 0.9 0.2]
 [0.2 0.1 0.8]]
```

这个 `(句长, D)` 的向量叠,就是真正喂进后续网络的输入。

## 3. 怎么量"含义有多近"?余弦相似度

有了每个词的向量,马上有个问题:怎么判断两个词"像不像"?需要一把**尺子**。最常用的是**余弦相似度(cosine similarity)**——它不看两个向量隔多远,只看它们**方向**有多一致:

```text
cos(u, v) = (u · v) / (|u| × |v|)

  · 结果在 -1 ~ 1 之间
  · 越接近 1  = 方向越一致 = 越「像」
  · 接近 0    = 互不相关
  · 接近 -1   = 方向相反
```

为什么用"方向"而不用"距离"?因为含义更多体现在朝向上。一个词向量整体放大两倍,意思没变,方向也没变,余弦相似度照样是 1。我们拿上一节那张手填的表,验证它确实做到了"猫离狗近、离桌子远":

```text
两两余弦相似度(越接近 1 越像):
  cos(猫, 狗)    = +0.990
  cos(桌子, 椅子) = +0.987
  cos(猫, 桌子)   = +0.303
  cos(狗, 椅子)   = +0.404
  -> 同类(猫·狗、桌子·椅子)高,跨类(猫·桌子)低。这张表确实带「含义」。
```

"只看方向、不看长短"这点可以直接验证——把"猫"的向量整体放大 5 倍,它和"狗"的相似度纹丝不动:

```text
  cos(猫, 狗)   = 0.990
  cos(猫×5, 狗) = 0.990   # 放大 5 倍,方向没变,相似度不变
```

有了这把尺子,就能做一件很实用的事:**给一个词,找出最像它的邻居**——这正是"近义词检索"的原理:

```text
  和『猫』最像的:  狗(+0.99)、椅子(+0.34)
  和『桌子』最像的:椅子(+0.99)、狗(+0.39)
```

## 4. 真实的表是"学"出来的:nn.Embedding

到这里有个尴尬的地方:上面那张漂亮的表,数字是我**手填**的。真实模型里,这几百维的向量从哪来?

答案是:**它们是训练出来的参数**。一开始全是随机数,模型在完成任务(比如反复猜下一个字)的过程中,用梯度下降一点点把它们调好。PyTorch 把这张"可学习的查找表"封装成一层:**`nn.Embedding(词表大小, 向量维度)`**。

```python
import torch.nn as nn
emb = nn.Embedding(4, 2)        # 4 个词,每个词 2 维
print(emb.weight.requires_grad) # True —— 这张表能被训练
emb(torch.tensor([0, 2]))       # 前向 = 按编号查行,和 E[i] 一模一样
```

它内部就是一个 `(V, D)` 矩阵,`requires_grad=True`(能求导)。前向就是"按编号查行",和第 2 节的 `E[i]` 没有区别,只是能自动求梯度。

**那随机初始化的向量,真能自己学出"同类相近"吗?** 我们设计一个极小的任务验证:告诉模型"猫~狗 是一类、桌子~椅子 是一类",让同类词的余弦相似度趋近 1、异类趋近 0,然后看随机向量能不能自己挪到位:

```text
开始训练(目标:同类相似度→1,异类→0):
  训练前:同类平均 cos = -0.988，异类平均 cos = -0.323   # 随机起步,一团乱
  step   0: loss=4.0557  同类 cos=-0.902  异类 cos=-0.098
  step 100: loss=0.0005  同类 cos= 0.978  异类 cos= 0.001
  step 200: loss=0.0001  同类 cos= 0.990  异类 cos= 0.000
  step 300: loss=0.0000  同类 cos= 0.993  异类 cos= 0.000
  训练后:同类平均 cos =  0.993，异类平均 cos =  0.000
```

短短 300 步,同类相似度从 -0.988 爬到 0.993,异类压到 0.000。再看每个词学到的 2 维坐标,会发现它们真的"分了堆":

```text
训练后,每个词学到的向量(随机初始化 → 梯度下降挪到位):
  猫:   [-0.13, +1.67]
  狗:   [-0.55, +2.22]    # 猫、狗 都在「上方」,挨在一起
  桌子: [-1.86, -0.29]
  椅子: [-1.41, -0.23]    # 桌子、椅子 都在「左下」,挨在一起

验证:cos(猫, 狗) = 0.987，cos(猫, 桌子) = -0.076
```

有一点要说清楚:真实 GPT 里**没有**"告诉它谁和谁是同类"这种监督。它只是在反复猜下一个字,这些含义向量是**顺带**学出来的副产品——但效果惊人地好。我们这里用了显式监督,只是为了让"随机→学出含义"这个过程看得见、摸得着。

## 本章小结

- **编号(token id)不是含义**:数值大小是我们随便编的,距离纯属巧合,不能拿来算"像不像"。
- **one-hot 也不行**:它解决了"大小是假的",但所有词两两等距(都是 √2),且维度随词表爆炸,依然没有含义。
- **Embedding = 一张可查的表**:`V` 行 × `D` 列,第 `i` 行就是第 `i` 号词的稠密向量。"查表"既是 `E[i]`,也等价于 `one-hot @ E`——本质是网络的一层。
- **余弦相似度**:只看方向、不看长短,范围 -1~1,是度量"含义有多近"的常用尺子。
- **nn.Embedding**:一张**可学习**的查找表。向量从随机起步,靠梯度下降训练,能自己学出"同类相近"。

至此,"文字 → 编号 → 含义向量"这条链就打通了:上一章把文字切成编号,这一章把编号查成向量。下一步,我们要让这些向量之间**互相看一看**、理解上下文——这就是 Transformer 的心脏:注意力机制。
