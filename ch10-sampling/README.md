# 第十章 · 采样与生成

> 模型每一步吐出的是 **logits**:对词表里每个字的打分。怎么把这堆打分变成「选定的那一个字」?
> 这一步叫**采样(sampling)**。它**不改变模型**,却深刻影响生成风格——是死板复读,还是灵动多样。
>
> 这一章讲清四种最常见的策略:**greedy、temperature、top-k、top-p**,
> 并把它们装到第九章那个会背诗的小 GPT 上,看真实文字的差别。

---

## 1. 起点:logits → 概率

模型输出的 logits 先过 softmax 变成概率分布。假设模型对 6 个候选字打分如下:

```text
原始概率 softmax(logits):
  月  0.529 █████████████████████
  光  0.321 ████████████
  霜  0.072 ██
  鸟  0.043 █
  风  0.032 █
  x  0.004
```

> 代码见 [`code/sampling_strategies.py`](code/sampling_strategies.py)(`x` 代表生僻/不通顺的字)

接下来的每种策略,都是在「怎么从这个分布里挑一个」上做文章。

---

## 2. greedy:永远选分最高的

最简单:直接取 `argmax`,永远选概率最大的字。

```python
np.argmax(logits)   # -> 月
```

**确定、可复现,但单调**。长文本里它极易陷入复读、绕圈——因为每一步的选择完全固定,
模型一旦进入某个循环就出不来。

---

## 3. temperature:调「胆量」的旋钮

把 logits **除以一个温度 T** 再 softmax。`T < 1` 拉大差距(分布更尖锐、更保守);
`T > 1` 拉平差距(更随机、更冒险);`T = 1` 就是原始分布。

```python
probs = softmax(logits / T)
```

同一组 logits,三个温度下的分布:

```text
temperature = 0.5:   月 0.716  光 0.263  霜 0.013  ...   (更尖,押注高分字)
temperature = 1.0:   月 0.529  光 0.321  霜 0.072  ...   (原始)
temperature = 1.5:   月 0.424  光 0.304  霜 0.112  ...   (更平,给冷门字机会)
```

温度越低越尖(押注高分字),越高越平(给冷门字机会)。这是最常用的「创意旋钮」。

---

## 4. top-k:只在前 k 名里挑

只保留分数最高的 `k` 个字,其余的概率清零(设 −∞ 再 softmax),然后在这 k 个里采样。

```python
keep = np.argsort(logits)[-k:]   # 分数最高的 k 个
logits[其余] = -np.inf           # 砍掉长尾
probs = softmax(logits)
```

```text
top-k = 3:   月 0.574  光 0.348  霜 0.078  鸟/风/x 全部=0(被排除)
```

作用是**砍掉长尾**,杜绝偶尔蹦出的离谱字。`k` 越小越保守。缺点:`k` 是个固定数,
不看分布形状——有时该多给点候选,有时该少给。

---

## 5. top-p(核采样):候选集大小自适应

按概率从高到低累加,凑够 `p`(比如 0.9)就停,只在这个**最小集合**里采样。

```python
order = np.argsort(probs)[::-1]      # 从高到低
csum = np.cumsum(probs[order])
cut = np.searchsorted(csum, p) + 1   # 累计刚好达 p 的位置
keep = order[:cut]
```

```text
top-p = 0.9:   月 0.574  光 0.348  霜 0.078  其余=0
```

妙处在**自适应**:模型很确定时(分布尖),少数几个字就凑够 p,候选集小;
模型犹豫时(分布平),要更多字才凑够 p,候选集大。这是目前最主流的策略,
也叫 **nucleus sampling(核采样)**。

---

## 6. 装到真 GPT 上,看真实差别

把这些策略接到第九章那个会背诗的小 GPT 上(快速重训几秒),同一起头字「白」,各生成 40 字:

> 代码见 [`code/generate_with_gpt.py`](code/generate_with_gpt.py)

```text
greedy(贪心,永远取最高分):
  白日依山尽，黄河入海流。欲穷千里目，更上一层楼。
锄禾日当午，汗滴禾下土。谁知盘中

temperature=0.3(低温,保守):
  白日依山尽，黄河入海流。欲穷千里目，更上一层楼。
锄禾日当午，汗滴禾下土。谁知盘中

temperature=1.0(原始分布):
  白日依山尽，黄河入海流。欲穷千里目，更上一层楼。
锄禾日当午，汗滴禾下土。谁知盘中

temperature=1.8(高温,放飞):
  白霜。举头思故乡。
春眠不觉晓，处处闻啼鸟。夜来风雨声，花落知多少。
白日海流。黄

top_k=5 + temperature=1.0(砍长尾):
  白日依山尽，黄河入海流。欲穷千里目，更上一层楼。
锄禾日当午，汗滴禾下土。谁知盘中
```

差别一目了然:greedy、低温、top-k 都稳稳背出正确诗句;唯独 **T=1.8 的高温**
开始串行、乱接别的诗、甚至不通顺(「白霜。举头思故乡」把两句诗缝在了一起)——
高温给了冷门字太多机会。

> 这个小模型几乎把语料背死了,所以 T=1.0 也很稳;真实大模型在 T=1.0 下就已经相当多样了。
> 但「高温更野、低温更稳」的方向是普适的。

## 本章小结

- **采样不改模型,只决定「怎么挑字」**,却直接左右生成风格。
- **greedy**:永远取最高分,确定但单调,易复读。
- **temperature**:`logits / T` 调胆量。`T<1` 更稳更尖,`T>1` 更野更平,`T=1` 为原始。最常用的创意旋钮。
- **top-k**:只在前 k 名里采,砍掉长尾烂字;k 固定,不看分布形状。
- **top-p(核采样)**:保留「累计概率达 p」的最小集合,候选数随模型把握度**自适应**;当前最主流。
- **实践常组合用**:先 top-k / top-p 砍长尾,再配适中 temperature 调多样性。这就是各家 API 暴露这些参数的原因。
